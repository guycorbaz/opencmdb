## What went well

- **The loop closed end to end**: real ping → ingest → reconcile → page → published image. The product thesis is now working code.
- **The "Reads bomb" compiled first try** (HRTB over GAT via the covariance trick) — a long-standing major risk retired.
- **`reconcile` is a pure function** (no clock, no SQL, no origin) — deterministically testable; the same for `build_view`, auth, and i18n (testable without a DB).
- **Discipline held throughout**: the frontier gate stayed green (the domain never touched a forbidden dependency), binary collation everywhere, honest empty states, **zero private data committed**.
- **Pragmatic, assumed trade-offs**: hand-authored CSS instead of the Tailwind pipeline; deny-by-default *seams* rather than premature full auth — no over-building.
- **Guy's note**: the stories ran smoothly, without snags; final acceptance will happen in **pre-production** testing (not per-story).

## Challenges / patterns

- **Recurring pattern — local vs CI drift (4×)**: local verification did not reproduce CI's environment or flags.
  - 3.3: parallel DB tests raced on `migrate!` on CI's single MariaDB → fixed with `DB_TEST_LOCK`.
  - 3.5: CI forbids unprivileged ICMP; `poll` opened the socket before the cancel check → fixed (check cancel first; gate the live scan).
  - 3.6: local `clippy` without `--all-targets` did not lint test code → an unused test import reddened CI.
  - 3.9: musl target added to the wrong toolchain → copy `rust-toolchain.toml` before `rustup target add`.
- **Task honesty (noted, no blame)**: in 3.5 a subtask was checked off before it was implemented (CIDR parsing); caught and corrected within the story. A habit to keep watching.
- **Collaboration friction (Guy's feedback)**: the assistant reflexively proposed a pause at the end of *every* story, which was annoying. Momentum should be the default; acceptance is a pre-production concern, not a per-story gate.

## Action items

1. **Harden the local verification gate to mirror CI** *(from the local-vs-CI pattern — Guy approved).*
   - Add a single entry point (e.g. `cargo xtask verify`) that runs exactly what CI runs: `cargo fmt --all --check`, `cargo clippy --workspace --all-targets --locked -- -D warnings`, `cargo xtask ci`, and `cargo test --workspace --locked` against a local MariaDB. Goal: `dev == CI`, so a green local run means a green CI run.
   - Owner: Dev (Amelia). Target: early in Epic 4.
2. **Drop the reflexive "pause?" at each story end** *(collaboration agreement).*
   - Keep momentum by default; validation happens in pre-production. Only check in at a genuine decision point or when Guy asks. Recorded to assistant memory so it does not recur.
   - Owner: Dev (Amelia). Effective immediately.
